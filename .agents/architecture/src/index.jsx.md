## Goal

Application entry point — the root file loaded by `index.html`. It creates the React root, configures routing via `react-router-dom` with file-based routes from generouted, wraps the app in Primer's `ThemeProvider` and `BaseStyles`, initializes the Storyboard core UI (toolbar, viewfinder, comments, inspector), installs the hash preserver for client-side navigation, and handles post-creation canvas redirects.

This is the single file that bootstraps the entire Storyboard prototyping app.

## Composition

### Imports

```jsx
import { routes } from './routes'                              // generouted file-based routes
import { ThemeSync } from '@dfosco/storyboard-react-primer'   // Primer ↔ Storyboard theme sync
import { installHashPreserver } from '@dfosco/storyboard-react/hash-preserver'
import { mountStoryboardCore } from '@dfosco/storyboard-core'
import storyboardConfig from '../storyboard.config.json'
```

### Canvas Redirect

Before anything renders, checks for a `?redirect=` query param (set by the canvas creation form). When Vite full-reloads after a new `.canvas.jsonl` is created, this redirects to the new canvas route:

```js
const redirectParam = new URLSearchParams(window.location.search).get('redirect')
if (redirectParam) {
    const base = (import.meta.env.BASE_URL || '/').replace(/\/$/, '')
    window.location.replace(base + redirectParam)
}
```

### Router Setup

```js
const router = createBrowserRouter(routes, {
    basename: import.meta.env.BASE_URL,
})
```

Uses `import.meta.env.BASE_URL` (from `VITE_BASE_PATH`) as the router basename, supporting branch deploys.

### Core Initialization

```js
installHashPreserver(router, import.meta.env.BASE_URL)
mountStoryboardCore(storyboardConfig, { basePath: import.meta.env.BASE_URL })
```

- **`installHashPreserver`** — Preserves URL hash (storyboard session state) across React Router navigations
- **`mountStoryboardCore`** — Mounts the Svelte-based toolbar, viewfinder, design modes, inspector, and comments UI

### Render Tree

```jsx
<StrictMode>
    <ThemeProvider colorMode="auto">
        <BaseStyles>
            <ThemeSync />
            <RouterProvider router={router} />
        </BaseStyles>
    </ThemeProvider>
</StrictMode>
```

- `ThemeProvider` + `BaseStyles` — Primer design system context
- `ThemeSync` — Synchronizes Primer's theme with Storyboard's theme state
- `RouterProvider` — React Router with generouted file-based routes

### CSS Imports

```jsx
import './reset.css'

```

## Notes

- The canvas redirect logic runs synchronously before React renders, using `window.location.replace` to avoid a flash of the wrong page.
- `import.meta.env.BASE_URL` is used in three places: router basename, hash preserver, and core mount — all must agree for branch deploys to work correctly.
