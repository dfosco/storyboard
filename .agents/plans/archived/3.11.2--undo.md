# Undo/Redo Queue for Canvas

## Problem

Canvas has no undo/redo support. Users need to freely undo and redo widget mutations (add, remove, edit, move) with Figma-like behavior: continuous edits coalesce into one step, selection is excluded, and any new mutation after an undo breaks the redo chain.

## Approach: Snapshot-based State History

Use a **past/present/future** snapshot model on the `localWidgets` array. This is simpler and more reliable than command-based undo, and naturally handles all mutation types uniformly.

```
past:    [snap0, snap1, ...]   ← undo pops from here
present: localWidgets          ← current live state
future:  [snapA, snapB, ...]   ← redo pops from here
```

- **On mutation**: push present → past, apply mutation, clear future
- **On undo**: push present → future, pop past → present
- **On redo**: push present → past, pop future → present

### Edit Coalescing (Figma behavior)

Continuous edits to the same widget (e.g. typing in a sticky note) coalesce into one undo step. A snapshot is only taken on the *first* edit after a different action type or widget target. This way, undo reverts the entire edit session, not each keystroke.

Track `lastAction = { type, widgetId }`:
- **add/remove**: always snapshot
- **move**: always snapshot (different widget or different action)
- **edit**: snapshot only if last action wasn't `('edit', sameWidgetId)`

### Server Sync on Undo/Redo

When undo/redo restores a widget array, persist immediately via `updateCanvas(name, { widgets })` (sends `widgets_replaced` event). Cancel any pending debounced saves to avoid race conditions.

### Limits & Resets

- Cap history at 100 entries (past + future)
- Clear history on canvas HMR reload (server state may have changed externally)
- Dev-only: keyboard shortcuts only fire when `import.meta.hot` is available

## Implementation Steps

### Step 1: Create `useUndoRedo` hook

- Refs for `past[]` and `future[]` arrays (avoid re-render on deep state)
- State counter for `canUndo`/`canRedo` booleans (triggers re-render for button disabled state)
- `snapshot(currentWidgets)` — deep-clones and pushes to past, clears future
- `undo(currentWidgets)` — returns previous state or null
- `redo(currentWidgets)` — returns next state or null
- `reset()` — clears both stacks (for HMR reloads)
- Max 100 snapshots in past (shift oldest when exceeded)

### Step 2: Integrate into CanvasPage

Wrap each mutation point:
1. `addWidget` — snapshot before appending server result
2. `handleWidgetRemove` — snapshot before filtering
3. `handleWidgetUpdate` — snapshot on first edit of a new sequence (coalesced)
4. `handleItemDragEnd` — snapshot before position update
5. Paste handlers — snapshot before appending pasted widget

Add undo/redo handlers:
- `handleUndo` — get previous state from hook, set localWidgets, persist to server
- `handleRedo` — get next state from hook, set localWidgets, persist to server
- Cancel pending debounced saves when undo/redo fires

Add keyboard shortcut listener:
- `Cmd+Z` → undo (only when `import.meta.hot` is truthy)
- `Cmd+Shift+Z` → redo (only when `import.meta.hot` is truthy)
- Skip when focus is in INPUT/TEXTAREA/contentEditable

Clear undo history when canvas reloads (track `trackedCanvas` changes).

### Step 3: Add UI to CanvasControls

Pass `canUndo`, `canRedo`, `onUndo`, `onRedo` props.

Layout: `[+] | [-] [100%] [+] | [↩] [↪]`

Buttons use same `.btn` class, disabled when stack is empty.

### Step 4: Style the button group

Add divider before undo/redo group. Use standard undo/redo arrow SVG icons.

## Edge Cases & Risks

- **Debounce race**: A pending debounced save could overwrite undo state. Solution: cancel debounce timer on undo/redo.
- **HMR reload**: External file changes invalidate history. Solution: clear stacks on canvas data change.
- **Memory**: 100 snapshots of widget arrays. Each snapshot is a small JSON array (~10-50 widgets). Safe.
- **Server errors on undo persist**: Log error, don't revert local state (optimistic).
- **JSX/source widgets**: These are NOT in the undo queue (they're component-based, not JSON widgets). Only `localWidgets` mutations are tracked.
